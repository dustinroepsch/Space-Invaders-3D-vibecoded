## Project Overview

A 3D space invaders game built with Bevy 0.18 in Rust (2024 edition). Features voxel-style 3D graphics, multi-wave progression, destructible barriers, a mystery ship, procedural audio, and a CRT post-process shader.

## Build Commands

- **Build:** `cargo build`
- **Run:** `cargo run`
- **Test:** `cargo test`
- **Run single test:** `cargo test <test_name>`
- **Check (fast compile check):** `cargo check`
- **Lint:** `cargo clippy`
- **Format:** `cargo fmt`

### Web (WASM) Build

- **Dev server:** `trunk serve` (localhost:8080)
- **Production build:** `trunk build` → outputs to `dist/`
- **WASM compile check:** `cargo check --target wasm32-unknown-unknown --features webgl2`

`Trunk.toml` sets `features = ["webgl2"]` automatically. The `getrandom` dependency under `[target.'cfg(target_arch = "wasm32")'.dependencies]` enables `rand::thread_rng()` on WASM via the Web Crypto API.

## Architecture

Single-binary Rust project using Bevy's plugin architecture. `main.rs` sets up the camera, lighting, starfield, and registers all plugins. Each gameplay concern is its own plugin module:

```
src/
  main.rs        — App entry, camera/lighting/starfield setup, plugin registration
  components.rs  — All shared types: marker components, resources, constants, GameState,
                   wave_config(), WaveConfig struct
  player.rs      — PlayerPlugin: ship mesh, A/D movement, Space to shoot
  enemy.rs       — EnemyPlugin: grid spawning, side-to-side + step-down movement, shooting
  bullet.rs      — BulletPlugin: velocity-based movement, out-of-bounds despawn
  collision.rs   — CollisionPlugin: distance-based hit detection for all bullet/entity pairs,
                   barrier damage, game-over trigger, wave-clear detection
  scoreboard.rs  — ScoreboardPlugin: HUD text, game-over/victory/wave-transition screens
  barrier.rs     — BarrierPlugin: four destructible voxel bunkers (6×5 block grid, 2 depth layers)
  mystery.rs     — MysteryPlugin: periodic mystery ship, score popup floating animation
  effects.rs     — EffectsPlugin: explosion particles, bullet trail particles, Lifetime system
  crt.rs         — CrtPlugin: fullscreen CRT scanline post-process (custom render graph node)
  sound.rs       — SoundPlugin: procedural SFX via SfxAsset/SfxDecoder, SoundQueue bus

assets/
  shaders/crt.wgsl — WGSL fragment shader for the CRT effect
```

## Key Patterns

### Game State Machine
`GameState` (in `components.rs`) drives all gameplay gating:
- `Playing` — normal gameplay; almost all systems use `.run_if(in_state(GameState::Playing))`
- `WaveTransition` — brief pause between waves
- `GameOver` / `Victory` — end states

### Wave System
`wave_config(wave: u32) -> WaveConfig` in `components.rs` returns per-wave tuning (speed, shoot interval, grid dimensions, z-offset). Up to 10 defined waves, then a hardcoded max config. `CurrentWave` and `EnemySpeed` are resources that drive this.

### Sound Bus
`SoundQueue(Vec<SoundKind>)` is a shared resource. Any system pushes a `SoundKind` to the queue; `SoundPlugin::play_sounds` drains it each `PostUpdate` and spawns `AudioPlayer` entities with `PlaybackSettings::DESPAWN`. All SFX are procedurally generated by `SfxAsset`/`SfxDecoder` — no audio files on disk.

### Mystery Ship Events
`MysteryShipKilledEvent` (a `Message`) is fired by `collision.rs` and handled by `mystery.rs` to spawn a floating score popup with a rise-and-fade animation.

### CRT Post-Process
`CrtSettings` is a marker component added to the camera. `CrtPlugin` inserts a custom render graph node between `Node3d::Tonemapping` and `Node3d::EndMainPassPostProcessing` using `FullscreenShader` + the `assets/shaders/crt.wgsl` fragment shader.

### Barrier Blocks
Each barrier block is a `Barrier { health: u8 }` entity. On bullet hit, health decrements and the material is replaced (`barrier_material(health)` in `barrier.rs`). At health 0 the block is despawned.

## Dependencies

- `bevy` 0.18 (`3d` feature) — game engine
- `rand` 0.8 — random enemy/mystery ship behavior
- `getrandom` 0.2 (WASM only, `js` feature) — enables `rand::thread_rng()` in browsers

## Game Controls

- **A / Left Arrow** — move left
- **D / Right Arrow** — move right
- **Space** — shoot
