## Project Overview

A 3D space invaders game built with Bevy 0.18 in Rust (2024 edition). Features voxel-style 3D graphics, multi-wave progression, destructible barriers, a mystery ship, procedural audio, and a CRT post-process shader.

## Build Commands

- **Build:** `cargo build`
- **Run:** `cargo run`
- **Test:** `cargo test`
- **Run single test:** `cargo test <test_name>`
- **Check (fast compile check):** `cargo check`
- **Lint:** `cargo clippy`
- **Format:** `cargo fmt`

### Web (WASM) Build

- **Dev server:** `trunk serve` (localhost:8080)
- **Production build:** `trunk build --release` ‚Üí outputs to `dist/` (uses `wasm-release` profile: `opt-level = "s"`, strips debuginfo)
- **WASM compile check:** `cargo check --target wasm32-unknown-unknown --features webgl2`

`Trunk.toml` sets `features = ["webgl2"]` automatically. The `getrandom` dependency under `[target.'cfg(target_arch = "wasm32")'.dependencies]` enables `rand::thread_rng()` on WASM via the Web Crypto API. `.cargo/config.toml` adds `-C target-feature=+reference-types` for the WASM target ‚Äî required by wasm-bindgen 0.2.100+.

## Architecture

Single-binary Rust project using Bevy's plugin architecture. `main.rs` sets up the camera, lighting, starfield, and registers all plugins. Each gameplay concern is its own plugin module:

```
src/
  main.rs        ‚Äî App entry, camera/lighting/starfield setup, plugin registration
  components.rs  ‚Äî All shared types: marker components, resources, constants, GameState,
                   wave_config(), WaveConfig struct
  player.rs      ‚Äî PlayerPlugin: ship mesh, A/D movement, Space to shoot
  enemy.rs       ‚Äî EnemyPlugin: grid spawning, side-to-side + step-down movement, shooting
  bullet.rs      ‚Äî BulletPlugin: velocity-based movement, out-of-bounds despawn
  collision.rs   ‚Äî CollisionPlugin: distance-based hit detection for all bullet/entity pairs,
                   barrier damage, game-over trigger, wave-clear detection
  scoreboard.rs  ‚Äî ScoreboardPlugin: HUD text (lives/wave/score/hi-score), game-over/wave-transition screens
  barrier.rs     ‚Äî BarrierPlugin: four destructible voxel bunkers (6√ó5 block grid, 2 depth layers)
  mystery.rs     ‚Äî MysteryPlugin: periodic mystery ship, score popup floating animation
  effects.rs     ‚Äî EffectsPlugin: explosion particles, bullet trail particles, Lifetime system
  crt.rs         ‚Äî CrtPlugin: fullscreen CRT scanline post-process (custom render graph node)
  sound.rs       ‚Äî SoundPlugin: procedural SFX via SfxAsset/SfxDecoder, SoundQueue bus

assets/
  shaders/crt.wgsl ‚Äî WGSL fragment shader for the CRT effect

index.html         ‚Äî Web entry point for Trunk: AudioContext patch (browser audio unlock),
                     loading screen (MutationObserver hides it when canvas appears),
                     mobile touch controls (shown only on touch devices via CSS media query)
```

## Key Patterns

### Game State Machine
`GameState` (in `components.rs`) drives all gameplay gating:
- `Playing` ‚Äî normal gameplay; almost all systems use `.run_if(in_state(GameState::Playing))`
- `WaveTransition` ‚Äî brief pause between waves
- `GameOver` ‚Äî end state (no Victory state; game loops endlessly like the original)

### Arcade-Accurate Gameplay
Matches the original 1978 arcade game:
- **5√ó11 formation** (55 aliens) every wave
- **Dynamic speed-up**: as aliens are killed, remaining aliens move faster (inverse scaling)
- **Row-based scoring**: top row (squid) = 30 pts, middle rows (crab) = 20 pts, bottom rows (octopus) = 10 pts
- **1 player bullet at a time**: can't fire again until current bullet hits or leaves screen
- **Column-based enemy shooting**: only the bottommost alive alien in each column can fire, max 3 enemy bullets on screen
- **3 lives**: player respawns with 2.5s invulnerability (blinking) when hit
- **Barriers respawn each wave** and are destroyed by aliens walking through them
- **Endless looping**: waves repeat with increasing difficulty (aliens start lower, move faster, fire more often)
- **High score** tracked across games within a session

### Wave System
`wave_config(wave: u32) -> WaveConfig` in `components.rs` returns per-wave tuning (base speed, shoot interval, z-offset). Formation is always 5√ó11. `InitialEnemyCount` tracks starting count for dynamic speed calculation. `CurrentWave` and `EnemySpeed` are resources that drive this.

### Sound Bus
`SoundQueue(Vec<SoundKind>)` is a shared resource. Any system pushes a `SoundKind` to the queue; `SoundPlugin::play_sounds` drains it each `PostUpdate` and spawns `AudioPlayer` entities with `PlaybackSettings::DESPAWN`. All SFX are procedurally generated by `SfxAsset`/`SfxDecoder` ‚Äî no audio files on disk.

### Mystery Ship Events
`MysteryShipKilledEvent` (a `Message`) is fired by `collision.rs` and handled by `mystery.rs` to spawn a floating score popup with a rise-and-fade animation.

### CRT Post-Process
`CrtSettings` is a marker component added to the camera. `CrtPlugin` inserts a custom render graph node between `Node3d::Tonemapping` and `Node3d::EndMainPassPostProcessing` using `FullscreenShader` + the `assets/shaders/crt.wgsl` fragment shader.

### Barrier Blocks
Each barrier block is a `Barrier { health: u8 }` entity. On bullet hit, health decrements and the material is replaced (`barrier_material(health)` in `barrier.rs`). At health 0 the block is despawned.

### Mobile Touch Controls (WASM only)
`index.html` exposes a `window.touchInput = { left, right, fire }` JS global. HTML buttons update it on `touchstart`/`touchend`. Each frame, `touch_input()` in `player.rs` reads this via `js_sys::Reflect` + `web_sys::window()`. On non-WASM builds the function is a no-op that returns `(false, false, false)`.

## Dependencies

- `bevy` 0.18 (`3d` feature) ‚Äî game engine
- `rand` 0.8 ‚Äî random enemy/mystery ship behavior

- `wasm-bindgen`, `web-sys` (`Window` feature), `js-sys` (WASM only) ‚Äî mobile touch input via `window.touchInput` JS global
